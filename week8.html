Interactive React Components

Now that we're diving into more React content, I thought rereading how components can respond to human actions would be a good idea. This article is a game making article from the React website. But it has a really simple and understandable section on how to make a component interactive.

In this example, they have a 3x3 grid filled with numbers. They want an X to appear in a square once the user clicks on it. They already made a Square component and initialized the game in the Board component. To add the on click function to each of the squares, you'd want to add it inside the html tag in the return statement like so: 

            return (
                <button className="square" onClick={ SOMEFUNCTION; }>
                    {this.props.value}
                </button>
            )

They're implementation wants the Square component to have a state. This allows the square to remember if it has been clicked. They add a constructor to set this.state to equal the value inside the square. It's also highlited that inside any component constructor needs to have a super(props) call. The revised version of the returned HTML looks like this: 

            return (
                <button className="square" onClick={() => this.setState({value: 'X'})}>
                    {this.state.value}
                </button>
            )

Some other neat things the article mentions is that by having this return statement in the render method, React will re-render the individual Square component when it is clicked. In addition, React will automatically update any child components of the Square due to the setState method being called.
