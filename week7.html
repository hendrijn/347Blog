Reinforcing React

The reading on how to set up React was a bit challenging. There was a lot of new information so I decided to take this week's blog post to find the React documentation on the subject. This walkthrough goes through Components and Props.

It starts off saying how components are functions that return React elements and accept props as a parameter. Props are the properties of the React component... something like name, color, size, etc. Here's an example of what a component would look like: 

            export function Greeting(props) {
                return <h1>Hello there, {props.name}!</h1>;
            }

Then the post talks about composing components and how to set values for things. This implementation is slightly different because they call ReactDom.render to display content on the screen. I prefer how we did it in the textbook reading though. To make Hello there, Chris appear, you can create another component function called App that creates the actual page: 

            function App() {
                return (
                    <Greeting name="Chris">;
                )
            }

This sends name="Chris" to Greeting as props. The post also reiterates the importance of never modifying the props themselves. I think I get more about what components are and how to use props within them. Figuring out how to dynamically change things using states might be a topic for next week.
